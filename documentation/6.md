# Разработка функционала обнаружения IP, Port, MAC и payload злоумышленника в ловушке vIED (для NTP/PTP и MMS атак)

## 1. Анализ и планирование

A3-A4 (Time Sync / PTP / SNTP Спуфинг)
Вектор: Атака на систему единого времени (сдвиг часов, подмена GPS-сигналов, подмена NTP/PTP). Реализация: В open_server.c мы поднимаем UDP-сокет на порту 123 (NTP).

- Чтобы извлечь MAC атакующего, мы распарсим ARP-таблицу Docker среды (чтением файла /proc/net/arp), чтобы преобразовать IP-адрес источника в MAC.
- Payload NTP-пакета будет конвертироваться в Hex-строку и так же записываться в поле value логгера (чтобы система Security Analytics/T-Pot имела сырые данные спуфинга).

## 2. Описание задачи

1) чтобы атаки на NTP/PTP записывали информацию что с ними пытаются сделать, payload и т.д.
2) логики обнаружения IP, port, mac атакующего

## 3. Особенности реализации

- Implement IP-to-MAC resolution via /proc/net/arp in C++ HoneypotLogger
- Implement Hex dumping of UDP payload for the NTP/PTP monitor thread

- Обнаружение MAC-адреса атакующего: По умолчанию, обычные recv() или MmsServerConnection_getClientAddress() возвращают IP-адрес. В Linux, чтобы узнать MAC-адрес устройства в локальной сети, мы можем прочитать системный файл /proc/net/arp. Написал C++ функцию в HoneypotLogger, которая будет принимать источник IP-адреса, читать /proc/net/arp и возвращать соответствующий MAC-адрес
- Логирование Payload NTP: В ntp_monitor_thread (который слушает порт 123), добал простую функцию hex_dump, которая будет брать принятый буфер (char buffer[1024]), конвертировать его в строку вида 1B 00 00 ... и записывать в поле value в  файл vied_events.json. Таким образом, будем видеть точное содержимое подделанного пакета.

2) Adding MAC resolution and NTP formatting hooks in HoneypotLogger C++ class
3) Updating C-API and HoneypotLogger Event method to format MAC and Payload Hex Strings
4) Implementing Logger_LogNtpEvent inside honeypot_logger_c_api.cpp


- MAC-адреса: В honeypot_logger.hpp теперь есть функция getMacFromIp, которая парсит системный файл /proc/net/arp. Во всех логах (vied_events.json) теперь появилось поле src_mac, которое будет автоматически подхватывать реальный MAC адрес Docker-сети (или UNKNOWN_MAC если его там нет).
- Payload NTP: Я реализовал C-API метод Logger_LogNtpEvent. В open_server.c он принимает сырой буфер (например атакующего) и пропускает его через функцию hexDump(). В логах в поле value теперь будет записываться строка вида "1B 00 00 00 00 00 ..." (hex-представление пакета), чтобы вы видели, что именно пытался просунуть хакер в систему времени.
